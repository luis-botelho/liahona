# 📚 Glossary

> **Projeto:** LIA — Local Intelligence for Assistance
>
> **Versão:** 1.0.0
>
> **Status:** Ativo

---

# Objetivo

Este documento reúne os principais termos utilizados no projeto LIA.

Seu objetivo é criar uma linguagem comum entre desenvolvedores, designers, colaboradores e demais participantes do projeto.

Sempre que possível, novos termos deverão ser adicionados a este documento.

---

# Termos Gerais

## LIA

Local Intelligence for Assistance.

Plataforma voltada para conectar pessoas, empresas e oportunidades dentro de uma comunidade.

---

## MVP

Minimum Viable Product.

Primeira versão do produto com funcionalidades suficientes para validar a proposta de valor.

---

## Epic

Grande funcionalidade que representa um objetivo de negócio.

Uma Epic é dividida em diversas User Stories.

---

## User Story

Descrição de uma necessidade do usuário.

Representa uma funcionalidade pequena e entregue com valor.

---

## Task

Atividade técnica necessária para concluir uma User Story.

---

## Bug

Comportamento inesperado da aplicação.

---

## Backlog

Lista priorizada de funcionalidades, melhorias, correções e ideias do projeto.

---

## Icebox

Área do Backlog destinada a ideias futuras que ainda não possuem prioridade de implementação.

---

## Milestone

Marco importante do desenvolvimento do projeto.

Representa um conjunto de entregas relacionadas.

---

## Label

Categoria utilizada para organizar Issues no GitHub.

---

# Engenharia

## ADR

Architecture Decision Record.

Documento utilizado para registrar decisões importantes de arquitetura.

---

## API

Application Programming Interface.

Camada responsável pela comunicação entre aplicações.

---

## REST

Estilo arquitetural utilizado para construção da API.

---

## Endpoint

URL responsável por disponibilizar um recurso específico da API.

---

## Migration

Alteração versionada da estrutura do banco de dados.

---

## ORM

Object-Relational Mapping.

Ferramenta responsável por mapear objetos para tabelas do banco.

---

## CI/CD

Integração Contínua e Entrega Contínua.

Processo automatizado para validar e publicar novas versões da aplicação.

---

## Deploy

Processo de disponibilização de uma nova versão da aplicação.

---

# UX e Design

## Wireframe

Representação estrutural de uma interface antes do design visual.

---

## Design System

Conjunto de princípios, padrões e componentes reutilizáveis utilizados na interface.

---

## Persona

Representação fictícia baseada no público-alvo da plataforma.

---

## User Flow

Fluxo percorrido pelo usuário para realizar uma tarefa.

---

## Responsividade

Capacidade da interface adaptar-se a diferentes tamanhos de tela.

---

# Produto

## KPI

Key Performance Indicator.

Indicador utilizado para medir desempenho e evolução do produto.

---

## Roadmap

Planejamento de evolução do produto ao longo do tempo.

---

## Produto

Conjunto de soluções desenvolvidas para gerar valor aos usuários.

Mais importante que funcionalidades é o problema que o produto resolve.

---

# Filosofia

## Progressive Engineering

Filosofia adotada pelo LIA.

O projeto evolui conforme a complexidade do problema.

Nunca implementamos soluções maiores do que a necessidade atual.

---

## Community First

Princípio que orienta o desenvolvimento do LIA.

As decisões do produto devem priorizar o benefício da comunidade antes de interesses técnicos ou métricas de vaidade.

---

# Evolução

Este glossário deverá crescer junto com o projeto.

Sempre que um novo conceito surgir, ele deverá ser documentado para manter uma linguagem comum entre todos os colaboradores.
