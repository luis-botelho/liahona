# 📦 Product Backlog — LIA

> "Todo grande produto começa com uma boa organização."

---

# 🎯 Objetivo

Esta pasta reúne todas as **Epics** do projeto LIA.

As Epics representam grandes objetivos de negócio e organizam o desenvolvimento do produto em funcionalidades de alto nível.

Nenhuma implementação deve começar sem estar vinculada a uma Epic.

---

# 🗺️ Fluxo de Desenvolvimento

Todo o desenvolvimento do LIA segue o seguinte fluxo:

```text
Epic
   ↓
User Stories
   ↓
Tasks
   ↓
Código
   ↓
Testes
   ↓
Review
   ↓
Merge
   ↓
Deploy
```

# 🌱 Filosofia

No LIA seguimos a filosofia:

> **Documentation First**

Planejamos antes.

Modelamos antes.

Documentamos antes.

Construímos depois.

---

# 🔄 Ciclo de Vida

Cada Epic passa pelos seguintes estados:

```
📋 Planned

↓

🚧 In Progress

↓

👀 Review

↓

✅ Done
```

---

# 💙 Nosso compromisso

Cada Epic deve possuir:

- Objetivo
- Valor de negócio
- Escopo
- Fora do escopo
- Critérios de aceite
- Dependências
- User Stories
- Definition of Done
- Impacto na documentação

Assim garantimos consistência durante toda a evolução do produto.
