# 📚 User Stories — LIA

> "Toda funcionalidade começa como uma necessidade de alguém."

---

# 🎯 Objetivo

Esta pasta reúne todas as **User Stories** do projeto LIA.

Cada Story descreve uma necessidade real do usuário e funciona como a ponte entre uma Epic e a implementação técnica.

Nenhuma Task deve existir sem estar vinculada a uma User Story.

---

# 🗺️ Fluxo

```text
Epic
   ↓
User Story
   ↓
Task
   ↓
Código
```

---

# 📦 Estrutura

```
docs/stories/

README.md

US-001-register-user.md
US-002-login.md
US-003-reset-password.md
...
```

---

# 📋 Template de uma Story

Cada Story deve conter:

- 🎯 Objetivo
- 👤 Persona
- 📝 História
- ✅ Critérios de Aceite
- 🚫 Fora do Escopo
- 🔗 Dependências
- 📚 Documentação afetada
- ✔️ Definition of Done

---

# 🌱 Filosofia

Uma Story deve ser pequena o suficiente para ser concluída em poucos dias.

Se estiver grande demais, deve ser dividida.
